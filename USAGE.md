# Руководство по использованию SmartAI

## Этап 1: Сбор данных

### Подготовка
1. Скомпилируйте и установите плагин на тестовый сервер
2. Пригласите игроков или используйте несколько аккаунтов

### Сбор легитных данных
```
/smartai start legit
```
- Играйте в обычный PvP
- Проведите минимум 50-100 боев
- Разные стили игры (меч, топор, лук)
- Разные игроки с разной чувствительностью мыши

```
/smartai stop
```

### Сбор данных читеров
```
/smartai start cheat
```
- Используйте популярные чит-клиенты (Wurst, LiquidBounce и т.д.)
- Включите KillAura/AimBot
- Разные настройки читов (скорость ротации, рандомизация)
- Проведите 50-100 боев

```
/smartai stop
```

### Рекомендации
- Соберите минимум 1000 боев каждого типа
- Чем больше данных, тем лучше модель
- Разнообразие важно: разные игроки, читы, настройки

## Этап 2: Обучение модели

### Установка зависимостей
```bash
pip install pandas scikit-learn numpy joblib flask
```

### Обучение
```bash
python train_model.py
```

Скрипт:
- Загрузит все CSV из `plugins/SmartAI/datasets/`
- Обучит Random Forest модель
- Покажет метрики (точность, precision, recall)
- Сохранит модель в `smartai_model.pkl`

### Оценка результатов
- Точность > 90% - отлично
- Точность 80-90% - хорошо
- Точность < 80% - нужно больше данных

## Этап 3: Запуск API сервера

```bash
python api_server.py
```

Сервер запустится на `http://localhost:5000`

Проверка:
```bash
curl http://localhost:5000/health
```

## Этап 4: Интеграция с плагином

### Настройка config.yml
```yaml
ai:
  enabled: true
  server-url: "http://localhost:5000"
  flag-threshold: 0.8
  sequence-length: 40
```

### Перезапуск плагина
```
/reload confirm
```

## Мониторинг

### Логи плагина
Проверяйте `logs/latest.log` на предупреждения и ошибки

### Метрики API
API сервер логирует все запросы и предсказания

### False Positives
Если легитные игроки банятся:
- Снизьте `flag-threshold` (например, до 0.9)
- Соберите больше легитных данных
- Переобучите модель

### False Negatives
Если читеры не детектятся:
- Соберите данные с этими конкретными читами
- Увеличьте `sequence-length`
- Попробуйте другую модель (LSTM, XGBoost)

## Продвинутые настройки

### Использование LSTM (нейросеть)
Для лучшего качества используйте TensorFlow/PyTorch с LSTM слоями

### Балансировка классов
Если данных читеров меньше, используйте:
- SMOTE (Synthetic Minority Over-sampling)
- Class weights в модели
- Data augmentation

### Ансамбли моделей
Комбинируйте несколько моделей для лучшей точности

## Troubleshooting

### Плагин не загружается
- Проверьте версию Paper (должна быть 1.21.4)
- Проверьте Java версию (должна быть 21)

### Данные не сохраняются
- Проверьте права на папку `plugins/SmartAI/datasets/`
- Проверьте логи на ошибки

### API не отвечает
- Проверьте, запущен ли `api_server.py`
- Проверьте firewall
- Проверьте URL в config.yml

### Низкая точность модели
- Соберите больше данных (минимум 1000 боев каждого типа)
- Проверьте баланс классов
- Попробуйте другие алгоритмы
