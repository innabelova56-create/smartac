# Улучшение точности модели SmartAI

## Проблема
Легитные игроки получают слишком высокие вероятности (до 0.50), что приводит к ложным срабатываниям.

## Решение

### 1. Переобучение модели с улучшенными параметрами

Модель была улучшена следующими изменениями:

- **Увеличено количество деревьев**: 100 → 200 (лучшая точность)
- **Увеличена глубина**: 10 → 15 (более сложные паттерны)
- **Добавлена балансировка классов**: `class_weight='balanced'`
- **Оптимизированы параметры разделения**: `min_samples_split=5`, `min_samples_leaf=2`

### 2. Улучшенная агрегация в API

API теперь использует более строгий подход:

- Взвешенная комбинация: среднее (50%) + 75-й перцентиль (30%) + медиана (20%)
- Дополнительная проверка: если >30% тиков имеют вероятность >0.7, финальная вероятность повышается
- Порог для "читера" повышен: 0.5 → 0.6

### 3. Как переобучить модель

#### Вариант 1: Быстрый способ (Windows)
```bash
retrain.bat
```

#### Вариант 2: Ручной способ
```bash
python train_model.py
```

### 4. Перезапуск API сервера

После переобучения нужно перезапустить API:

1. Остановите текущий процесс API (Ctrl+C в терминале)
2. Запустите снова:
```bash
python api_server.py
```

### 5. Сбор дополнительных данных

Для лучшей точности рекомендуется:

1. **Больше легитных данных**: Соберите данные от разных легитных игроков
   ```
   /smartai start <player> legit
   ```

2. **Разнообразие читов**: Соберите данные от разных типов читов (KillAura, AimAssist, etc.)
   ```
   /smartai start <player> cheat
   ```

3. **Минимум 1000 тиков на сессию**: Чем больше данных, тем лучше

### 6. Проверка результатов

После переобучения проверьте:

1. **Точность на тестовой выборке**: Должна быть >90%
2. **Легитные игроки**: Вероятность должна быть <0.3
3. **Читеры**: Вероятность должна быть >0.8

### 7. Настройка порогов

Если модель все еще дает ложные срабатывания, измените пороги в `config.yml`:

```yaml
alerts:
  threshold: 0.75  # Повысьте до 0.80 или 0.85

violation:
  threshold: 40    # Повысьте до 50 или 60
```

### 8. Мониторинг

Используйте команды для мониторинга:

```
/smartai prob <player>  - Отслеживание вероятности в реальном времени
/smartai suspects       - Просмотр всех подозрительных игроков
```

## Дополнительные советы

1. **Баланс данных**: Убедитесь что у вас примерно одинаковое количество легитных и читерских данных
2. **Качество данных**: Собирайте данные во время реального PvP, а не на спавне
3. **Регулярное обновление**: Переобучайте модель каждые 1-2 недели с новыми данными
4. **Тестирование**: Всегда тестируйте новую модель перед использованием на продакшене

## Техническая информация

### Признаки (Features)
- `delta_yaw`, `delta_pitch` - Изменение ротации
- `accel_yaw`, `accel_pitch` - Ускорение ротации
- `jerk_yaw`, `jerk_pitch` - Рывок (изменение ускорения)
- `gcd_error_yaw`, `gcd_error_pitch` - Ошибка GCD (Greatest Common Divisor)

### Алгоритм
- **Random Forest Classifier** - Ансамбль из 200 деревьев решений
- **Stratified Split** - Сохранение пропорций классов при разделении
- **Balanced Weights** - Автоматическая балансировка классов

### Метрики
- **Accuracy** - Общая точность
- **Precision** - Точность для класса "читер"
- **Recall** - Полнота для класса "читер"
- **F1-Score** - Гармоническое среднее precision и recall
