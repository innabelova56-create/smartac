# Система накопления нарушений (Violation System)

## Как это работает

Теперь плагин использует систему накопления уверенности - чем дольше игрок демонстрирует читерское поведение, тем выше растет его вероятность.

### Для читеров (вероятность > 0.75):

1. **Каждый удар с высокой вероятностью увеличивает буфер**
   - Формула: `buffer += (probability - threshold) * multiplier`
   - Пример: вероятность 0.80, порог 0.75, множитель 100
   - Увеличение: `(0.80 - 0.75) * 100 = 5.0`

2. **Буфер повышает отображаемую вероятность**
   - Бонус: `min(buffer / 100, 0.3)` (максимум +0.3)
   - Пример: буфер 50 → бонус +0.25
   - Итоговая вероятность: `0.68 + 0.25 = 0.93`

3. **Результат: чем дольше бьет читер, тем выше вероятность**
   - 1-10 ударов: 0.68-0.70
   - 20-30 ударов: 0.75-0.85
   - 50+ ударов: 0.90-0.99

### Для легитных игроков (вероятность < 0.1):

1. **Каждый удар с низкой вероятностью уменьшает буфер**
   - Формула: `buffer = max(0, buffer - decay)`
   - Пример: decay = 0.35
   - Уменьшение: `buffer - 0.35`

2. **Низкий буфер понижает отображаемую вероятность**
   - Штраф: `min(buffer / 200, 0.15)` (максимум -0.15)
   - Пример: буфер 10 → штраф -0.05
   - Итоговая вероятность: `0.40 - 0.05 = 0.35`

3. **Результат: чем дольше бьет легит, тем ниже вероятность**
   - 1-10 ударов: 0.35-0.40
   - 20-30 ударов: 0.25-0.30
   - 50+ ударов: 0.10-0.20

### Для средних значений (0.1 - 0.75):

- Буфер не меняется
- Вероятность остается как есть
- Это зона неопределенности

## Настройка в config.yml

```yaml
alerts:
  threshold: 0.75  # Порог для начала накопления буфера

violation:
  multiplier: 100.0  # Множитель увеличения буфера (выше = быстрее растет)
  decay: 0.35        # Скорость уменьшения буфера (выше = быстрее падает)
```

### Рекомендуемые значения:

**Агрессивная настройка (быстрое обнаружение):**
```yaml
alerts:
  threshold: 0.70
violation:
  multiplier: 150.0
  decay: 0.5
```

**Сбалансированная (по умолчанию):**
```yaml
alerts:
  threshold: 0.75
violation:
  multiplier: 100.0
  decay: 0.35
```

**Консервативная (меньше ложных срабатываний):**
```yaml
alerts:
  threshold: 0.80
violation:
  multiplier: 80.0
  decay: 0.25
```

## Примеры работы

### Пример 1: Читер с KillAura

```
Удар 1:  prob=0.68, buffer=0    → display=0.68
Удар 5:  prob=0.70, buffer=15   → display=0.85
Удар 10: prob=0.68, buffer=30   → display=0.98
Удар 20: prob=0.70, buffer=50   → display=0.99
```

Результат: Через 20 ударов вероятность выросла с 0.68 до 0.99

### Пример 2: Легитный игрок

```
Удар 1:  prob=0.40, buffer=0    → display=0.40
Удар 5:  prob=0.35, buffer=0    → display=0.35
Удар 10: prob=0.08, buffer=0    → display=0.08
Удар 20: prob=0.05, buffer=0    → display=0.05
```

Результат: Через 20 ударов вероятность упала с 0.40 до 0.05

### Пример 3: Читер пытается скрыться

```
Удар 1-10:  prob=0.80, buffer=50  → display=0.99
Удар 11-20: prob=0.30, buffer=50  → display=0.30 (буфер не меняется)
Удар 21-30: prob=0.80, buffer=100 → display=0.99
```

Результат: Буфер сохраняется, читер не может "обмануть" систему

## Преимущества новой системы

1. **Накопление уверенности** - чем дольше читер бьет, тем выше вероятность
2. **Защита от выбросов** - один случайный высокий удар не вызовет бан
3. **Адаптация к легитным игрокам** - вероятность понижается со временем
4. **Защита от манипуляций** - читер не может "сбросить" буфер средними ударами

- Зеленый (< 0.6): легит
- Оранжевый (0.6-0.8): подозрительно
- Красный (>= 0.8): читер

## Технические детали

### Формулы:

**Увеличение буфера (читер):**
```
if probability > threshold:
    buffer += (probability - threshold) * multiplier
    bonus = min(buffer / 100, 0.3)
    display = min(probability + bonus, 1.0)
```

**Уменьшение буфера (легит):**
```
if probability < 0.1:
    buffer = max(0, buffer - decay)
    penalty = min(buffer / 200, 0.15)
    display = max(probability - penalty, 0.0)
```

**Средние значения:**
```
if 0.1 <= probability <= threshold:
    display = probability  # без изменений
```

### Ограничения:

- Максимальный бонус: +0.30
- Максимальный штраф: -0.15
- Минимальный буфер: 0.0
- Максимальная вероятность: 1.0
- Минимальная вероятность: 0.0
