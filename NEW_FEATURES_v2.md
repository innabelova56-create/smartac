# Новые возможности SmartAI v2

## Что изменилось

### 1. Ускоренное обновление голограммы
- Обновление каждый тик (было каждые 5 тиков)
- Голограмма теперь мгновенно следует за игроком
- Нет задержек и отставания

### 2. Показ всех оранжевых и красных алертов
- Порог алертов снижен с 0.75 до 0.60
- Теперь показываются:
  - Оранжевые (0.6-0.8) - подозрительно
  - Красные (>= 0.8) - читер
- В консоль пишутся только красные (>= 0.8)

### 3. Вывод в чат для `/smartai prob`
- Команда `/smartai prob <player>` теперь пишет в чат каждый удар
- Формат: `PlayerName Удар #10: 85.3%`
- Цвет процента зависит от вероятности:
  - Зеленый (< 0.6)
  - Оранжевый (0.6-0.8)
  - Красный (>= 0.8)

### 4. GUI для подозрительных игроков
- Команда `/smartai suspects` открывает GUI
- Показывает всех онлайн игроков с данными
- Клики:
  - ЛКМ - телепорт к игроку
  - ПКМ - телепорт + режим спектатора

## Использование

### Отслеживание игрока в чате

```
/smartai prob MirsTam
```

Вы увидите в чате каждый удар:
```
MirsTam Удар #1: 68.5%
MirsTam Удар #2: 72.3%
MirsTam Удар #3: 85.1%
```

Повторная команда отключает отслеживание:
```
/smartai prob MirsTam
```

### Просмотр подозрительных через GUI

```
/smartai suspects
```

Откроется меню с головами всех онлайн игроков. Кликните:
- ЛКМ - телепортироваться к игроку
- ПКМ - телепортироваться + включить спектатор

### Алерты в чате

Теперь вы будете видеть алерты для всех подозрительных:

```
HAC | AC MirsTam провалил AimCheck (68.5%)  - оранжевый
HAC | AC GanryPro провалил AimCheck (92.1%) - красный
```

## Настройка

### Изменить порог алертов

В `config.yml`:
```yaml
alerts:
  threshold: 0.60  # Показывать от 60%
```

Можно повысить до 0.70 или 0.75 если слишком много алертов.

### Изменить скорость обновления голограммы

В коде `HologramManager.java`:
```java
.runTaskTimer(plugin, 20L, 1L); // 1L = каждый тик
```

Можно изменить на 2L или 3L если есть лаги.

## Команды

```
/smartai prob <player>     - Отслеживание в чате (toggle)
/smartai suspects          - Открыть GUI с подозрительными
/smartai alerts            - Включить/выключить алерты
/smartai start <...>       - Начать сбор данных
/smartai stop <...>        - Остановить сбор
/smartai datastatus        - Статус сбора
/smartai reload            - Перезагрузить конфиг
```

## Примеры использования

### Сценарий 1: Проверка подозрительного игрока

1. Включите отслеживание: `/smartai prob Cheater123`
2. Наблюдайте за ударами в чате
3. Если вероятность растет - откройте GUI: `/smartai suspects`
4. Кликните ПКМ на голову игрока для телепорта в спектаторе
5. Наблюдайте за игроком

### Сценарий 2: Мониторинг сервера

1. Включите алерты: `/smartai alerts`
2. Играйте как обычно
3. При появлении алерта откройте GUI: `/smartai suspects`
4. Проверьте подозрительного игрока

### Сценарий 3: Сбор данных

1. Найдите читера
2. Начните сбор: `/smartai start Cheater123 cheat`
3. Пусть играет 5-10 минут
4. Остановите: `/smartai stop Cheater123`
5. Переобучите модель: `python train_model.py`

## Технические детали

### Формат алертов

```
HAC | AC <player> провалил AimCheck (<percent>%)
```

- HAC | AC - красный градиент (жестко задан)
- player - белый
- провалил - голубой (aqua)
- AimCheck - белый
- percent - цвет зависит от вероятности

### GUI

- Размер: 54 слота (6 рядов)
- Первые 45 слотов - головы игроков
- Последний ряд - серое стекло (заполнитель)
- Обновление: при открытии

### Отслеживание в чате

- Хранится в Map<UUID, UUID> (кто кого отслеживает)
- Вывод при каждой проверке (каждые sample-interval ударов)
- Автоматически отключается при выходе
